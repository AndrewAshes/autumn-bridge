## This session — what shipped + what's open

### URGENT prod fix shipped (PR #14)
Square sync was failing in prod since 6/17 08:30 — "Upsert failed at chunk 5500: statement timeout" → entire sync aborted → stale data (Ethan's Gresham short-by-$79.25 was the visible symptom).

**Root cause**: 250-row chunk exceeded Postgres `statement_timeout` as order volume grew. The throw aborted the whole run.

**Fix**: chunk size 250 → 100, per-chunk try/catch + one retry at half size (50), failed chunks no longer abort, new `chunksFailed` field on SyncSummary. Cron handler distinguishes **partial sync** (`phase=partial_upsert`, severity=warn) from **full failure** (`phase=run`, severity=critical) in /audit/errors. ADDITIVE only.

### Cash freshness refinement to PR #13
Before merge, refined the green/stale logic to key on `last_synced_at` not `last_transaction_at`. A truck on an off-day or café closed for a holiday now stays GREEN if the sync ran — was previously false-flagging as stale. Spec doc updated.
